JDBC DRIVERS

  • JDBC drivers implement the defined interfaces in the JDBC API, for interacting with the database server.
  • The java.sql package contains interfaces and classes of JDBC API.
  • To interact with the Database we need drivers. There are four types of drivers:
    1. JDBC - ODBC Bridge Driver
    1. JDBC- Native API
    1. JDBC - Network protocol API
    1. JDBC - Thin / Universal Driver

Type-1: JDBC - ODBC Bridge Driver

  • ODBC stands for Open Database Connectivity.
  • ODBC is developed by Microsoft. For JDBC, ODBC is acting as a bridge

    Architecture →

  • As Microsoft products works on Windows operating System, ODBC is O.S dependent.
  • After Java7, this driver is completely discontinued (not deprecated).
  • Performance degraded because JDBC method call is converted into the ODBC function calls.

Type-2: JDBC - Native API

  • The Native API driver uses the client-side libraries of the database. The driver converts JDBC method calls into native calls of the database API. It is not written entirely in java (mainly written in C/C++).

    Architecture →

  • Performance is little bit upgraded than JDBC-ODBC driver.
  • This removes O.S dependency, so can work with any Operating System
  • The Native driver needs to be installed on the each client machine.

Type-3: JDBC - Network protocol API

  • This Protocol uses middleware (server) that converts JDBC calls into vendor-specific database protocol.

    Architecture →

  • No client side library is required because of application server.
  • Maintenance of Network Protocol driver becomes costly because it requires database-specific coding to be done in the middle tier.
  • It also needs to install on every client machine, but while connecting with the server it automatically installs the driver.

Type-4: JDBC - Thin Driver

  • The thin driver converts JDBC calls directly into the vendor-specific database protocol. That is why it is known as thin driver. It is fully written in Java language.

    Architecture →

  • Better Performance than all the Drivers.
  • No installation on every client machine.
